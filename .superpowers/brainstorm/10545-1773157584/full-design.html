Plugin API v2 — Full Design Review

Review each section. Does this capture everything we discussed?

1

Monorepo Structure

npm workspaces + turborepo. packages/extension, packages/plugin-api, and packages/plugin-* for each built-in plugin. Types package published to NPM from same repo. Plugins graduate to own repos once API is stable.

2

Distribution

Plugins are VS Code extensions with extensionDependencies on CodeGraphy. VS Code handles activation order, install/uninstall, and marketplace. No custom infrastructure.

3

codegraphy.json Manifest

Separate manifest for CodeGraphy config — API version, tier, capabilities, rules, colors, filters, webview contributions. Published JSON schema for autocomplete.

4

Plugin Lifecycle

5 phases: Discovery → onLoad → onWorkspaceReady → onWebviewReady → onUnload. 3 recurring hooks: onPreAnalyze, onPostAnalyze, onGraphRebuild. Disposable auto-cleanup on everything.

5

Event System

30+ typed events across 6 categories: Graph Interaction (12), Analysis (4), Workspace (6), Views (6), Plugin Ecosystem (6), Timeline (4). All fully typed via EventPayloads interface.

6

Two-Tier Rendering

Tier 1: Structured decorations (badges, borders, tooltips, labels, colors, grouping). Tier 2: Webview JS/CSS injection with versioned API object, scoped containers, drawing helpers, auto-cleanup.

7

Core API Surface

CodeGraphyAPI with: events (on/once/off), decorations (decorateNode/Edge), graph queries (getGraph/getNode/getNeighbors), registration (views, commands, context menus), webview comms, utilities.

8

Versioning

Strict semver for Tier 1. Separate webview API version for Tier 2. Types package version locked to core. Deprecations last 2 major versions. Compatibility check on registration.

9

Registration Flow

vscode.extensions.getExtension → exports → registerPlugin(). Full code example in design doc showing event subscription, context menus, views, and decorations.

10

Built-In Migration

All 5 plugins migrate to registerPlugin(). Stay in monorepo for now. Each gets own codegraphy.json. Dogfood the API. Graduate to own repos once stable.

11

Validated Plugin Ideas

8 Tier 1 plugins (Metrics, Git Blame, Tags, Dep Chain, Issues, Actions, Architecture, Themes) and 4 Tier 2 plugins (Annotations, Previews, Sparklines, Treemap) — all achievable with the designed API surface.

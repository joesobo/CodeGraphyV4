Tier 2 Stability Strategies

How to make webview injection as stable as possible:

  • Versioned API object — codegraphyGraph.apiVersion — plugins declare target version, we can warn on mismatch
  • Scoped containers — each plugin gets its own DOM container for overlays, namespaced CSS
  • Auto-cleanup — like Obsidian's register() pattern, everything a plugin registers is automatically torn down on unload
  • Abstract drawing primitives — helpers like drawBadge(), drawProgressRing() that work regardless of canvas/SVG/WebGL
  • Capability declaration — plugins list Tier 2 features they use in manifest; deprecation warnings surface early
  • Sandboxed message channel — plugin webview code communicates with extension host via a typed message bus, not global state

Plugin Lifecycle, Events & Versioning

The backbone of a stable, Obsidian-style plugin system

Plugin Lifecycle

Modeled after Obsidian's onload/onunload + VS Code's activate/deactivate

1. Discovery
VS Code activates plugin extension. Plugin calls codegraphy.registerPlugin()
2. onLoad
Core calls plugin.onLoad(api). Plugin receives full API surface. Registers commands, views, decorators.
3. onWorkspaceReady
Workspace analyzed, graph data available. Plugin can query nodes/edges, attach decorations.
4. onWebviewReady
Webview is visible. Tier 2 plugins can now inject JS/CSS. Called again if webview is reopened.
5. onUnload
Plugin deactivating. All registered disposables auto-cleaned. No manual teardown needed.
Auto-cleanup pattern (from Obsidian): Every api.register*() call returns a Disposable. Plugins can manually dispose, or let the core auto-dispose everything when onUnload fires. Event subscriptions, commands, views, decorators, webview contributions — all auto-cleaned.

Recurring Hooks (called multiple times)

onPreAnalyze(files)
Before each analysis pass. Build indexes, prepare state. Receives full file list.
onPostAnalyze(graph)
After analysis. Receive full graph data. Attach decorations, compute metrics, update badges.
onGraphRebuild(graph)
Graph rebuilt without re-analysis (rule toggle, plugin toggle). Re-apply decorations.

Versioning Strategy

Core API Version

The CodeGraphy extension exports a versioned API. Follows semver strictly.

Major (2.0 → 3.0) — Breaking changes to existing hooks
Minor (2.1 → 2.2) — New events, decorations, APIs
Patch (2.1.0 → 2.1.1) — Bug fixes only

Plugin Manifest

Plugins declare compatibility in their package.json:

{
  "codegraphy": {
    "apiVersion": "^2.0.0",
    "tier": 1,
    "capabilities": [
      "decorations",
      "events",
      "contextMenu",
      "views"
    ]
  }
}
Tier 1 Stability
Strong guarantees. Structured decorations + events follow strict semver. Deprecations last 2 major versions with console warnings and migration guides.
Tier 2 Stability
Best-effort. Webview API versioned separately. Breaking changes possible on minor versions. Plugins declare webviewApiVersion — core warns on mismatch.
Compatibility Check
On registration, core checks apiVersion range. Future version → error. Deprecated version → warning + shim. Unsupported → reject with message.

What It Looks Like In Practice

Third-Party Plugin (their extension)
export async function activate(ctx) {
  // Wait for CodeGraphy to be ready
  const cg = await getCodeGraphyAPI();

  // Register plugin — core calls onLoad(api)
  cg.registerPlugin({
    id: 'codegraphy-metrics',
    name: 'Code Metrics',
    version: '1.0.0',
    apiVersion: '^2.0.0',

    onLoad(api) {
      // Subscribe to events
      api.on('analysis:completed', (e) => {
        for (const node of e.graph.nodes) {
          api.decorateNode(node.id, {
            badge: { text: `${node.imports}`, color: '#f07070' },
            tooltip: { sections: ['Metrics...'] }
          });
        }
      });

      // Add context menu item
      api.registerContextMenuItem({
        label: 'View Metrics',
        when: 'node',
        action: (node) => showMetricsPanel(node)
      });

      // Register a custom view
      api.registerView({ /* ... */ });
    },

    onUnload() { /* auto-cleaned */ }
  });
}
Tier 2 Plugin (with webview code)
// extension host side
cg.registerPlugin({
  id: 'codegraphy-sparklines',
  tier: 2,
  apiVersion: '^2.0.0',
  webviewApiVersion: '^1.0.0',

  // JS/CSS to inject into webview
  webviewContributions: {
    scripts: ['dist/webview.js'],
    styles: ['dist/webview.css'],
  },

  onLoad(api) {
    // Send data to webview module
    api.on('analysis:completed', (e) => {
      api.sendToWebview({
        type: 'SPARKLINE_DATA',
        data: computeSparklines(e.graph)
      });
    });
  }
});

// ---- webview side (dist/webview.js) ----
codegraphyGraph.onMessage('SPARKLINE_DATA', (d) => {
  sparklineData = d.data;
});

codegraphyGraph.registerNodeRenderer(
  'sparkline',
  (node, ctx, helpers) => {
    helpers.drawDefault(node, ctx); // base render
    drawSparkline(ctx, sparklineData[node.id]);
  }
);
